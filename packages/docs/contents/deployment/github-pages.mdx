# GitHub Pages Deployment

The current deployment workflow publishes the static gallery build from `apps/web/dist/` to GitHub Pages.

## Repository Settings

In GitHub, configure:

- **Pages source**: GitHub Actions.
- **Workflow**: `.github/workflows/deploy.yml`.
- **Branch**: `main`.

The workflow declares the `github-pages` environment and deploys through `actions/deploy-pages`.

## Local Build Equivalent

To reproduce the main build locally:

```bash
pnpm install
pnpm run photos:standardize
pnpm run build:manifest
SKIP_PRECHECK=1 pnpm build
```

The deployable static files will be in:

```plain
apps/web/dist/
```

## Workflow Output

During CI, the workflow also copies the build output into:

```plain
web/
```

This directory is committed back to the repository together with `photos/**` and `apps/web/src/data/photos-manifest.json`, so it can be used by static hosts that deploy from a repository subdirectory.

## Custom Domain

The gallery site configuration currently points to:

```plain
https://photo.jackyw.cn
```

Keep `config.json` and your GitHub Pages custom-domain/DNS settings aligned so canonical URLs, OpenGraph metadata, and sitemap output use the deployed domain.
