# GitHub Action Deployment

This repository includes a production deployment workflow at `.github/workflows/deploy.yml`. It is designed for a static SPA deployment and currently publishes `apps/web/dist/` to GitHub Pages.

The workflow runs on pushes to `main` when these paths change:

- `photos/**`
- `apps/**`
- `packages/**`
- `package.json`
- `pnpm-lock.yaml`
- `builder.config.ts`
- `site.config.ts`

It can also be started manually through `workflow_dispatch`.

## Build Steps

The workflow performs the following steps:

1. Check out the repository.
2. Install pnpm `10.19.0`.
3. Set up Node.js `24` with pnpm cache.
4. Install dependencies with `pnpm install`.
5. Run `pnpm run photos:standardize`.
6. Run `pnpm run build:manifest`.
7. Run `pnpm run build` with `SKIP_PRECHECK=1` to avoid rebuilding the manifest a second time.
8. Verify that `apps/web/dist/` exists.
9. Copy `sitemap.xml` to `googlesitemap.xml` when it exists.
10. Copy the built web output to the root `web/` directory.
11. Commit generated photo, manifest, and `web/` output changes.
12. Upload `apps/web/dist/` as a GitHub Pages artifact.
13. Deploy the artifact to GitHub Pages.

## Permissions

The workflow needs:

- `contents: write` to commit generated output back to the repository.
- `pages: write` to publish GitHub Pages.
- `id-token: write` for GitHub Pages deployment authentication.

## Output Targets

The primary deployment artifact is:

```plain
apps/web/dist/
```

The workflow also mirrors that static output to:

```plain
web/
```

This makes it possible to point another static host, such as Cloudflare Pages, at `web/` while GitHub Pages continues to deploy from `apps/web/dist/`.

## Notes

- Photo source files under `photos/` remain private copyrighted works and are only committed as part of this specific personal gallery workflow.
- `photos/incoming/` is excluded from the manifest by the current `builder.config.ts`; run `pnpm run photos:standardize` or move files into a real category before publishing.
- If the web build is run locally after `pnpm run build:manifest`, set `SKIP_PRECHECK=1` only when you intentionally want to skip the manifest precheck.
