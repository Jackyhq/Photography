# Jacky's Photography

Jacky's Photography is a customized personal photo gallery based on [Afilmory](https://github.com/Afilmory/afilmory). It keeps the core static-gallery workflow from Afilmory while adding project-specific automation, deployment, photo standardization, map exploration, and presentation refinements for [photo.jackyw.cn](https://photo.jackyw.cn).

> Photos under `photos/` are Jackyhq's personal works and are not part of the open-source license. Do not reuse, redistribute, or publish them without explicit written permission.

## What It Builds

The project builds a pure client-side React SPA. Photo metadata is generated before the frontend build and embedded as `photos-manifest.json`, so the deployed gallery can run on static hosting without an application server.

Core user-facing features:

- Responsive masonry photo gallery.
- Fullscreen photo viewer with WebGL zoom and pan.
- EXIF, histogram, camera, lens, tag, Live Photo, HDR, and Fujifilm recipe display.
- Command palette and gallery controls for searching, sorting, and changing layout.
- MapLibre-based location explorer for photos with GPS metadata.
- OpenGraph, sitemap, RSS utilities, and static deployment output.

## Storage Providers

`@afilmory/builder` reads photos through a storage-provider abstraction. The current project uses the local provider:

```ts
import { defineBuilderConfig } from '@afilmory/builder'

export default defineBuilderConfig(() => ({
  storage: {
    provider: 'local',
    basePath: './photos',
    baseUrl: 'https://photos3.jackyw.cn/photos/',
    excludeRegex: '^incoming($|/.*)',
  },
}))
```

Supported providers:

- `local`: reads local photo directories; useful for this repository, development, and self-hosted deployments.
- `s3`: reads S3-compatible object storage with optional custom domains and network tuning.
- `github`: reads photos from a GitHub repository or subdirectory.
- `eagle`: reads an Eagle 4 library and can filter by tags or folders.

## Development

```bash
pnpm install
pnpm run build:manifest
pnpm dev
```

Useful commands:

```bash
pnpm run photos:standardize
pnpm run build:manifest -- --force
pnpm run build:manifest -- --force-thumbnails
pnpm run build:manifest -- --force-manifest
pnpm run build:manifest -- --config
pnpm build
pnpm docs:dev
pnpm docs:build
pnpm --filter web type-check
```

`pnpm dev` and `pnpm build` run the web precheck, which generates the manifest unless `SKIP_PRECHECK=1` is set.

## Photo Workflow

1. Put new photos in `photos/incoming/` or directly in a category folder under `photos/`.
2. Run `pnpm run photos:standardize` to rename incoming files from EXIF timestamps and move them into category folders.
3. Run `pnpm run build:manifest` to scan photos, extract metadata, generate thumbnails, and update `apps/web/src/data/photos-manifest.json`.
4. Run `pnpm dev` for local preview or `pnpm build` for static production output.

## Deployment

The repository includes `.github/workflows/deploy.yml`. On pushes to `main` that touch photos, app code, packages, or project configuration, the workflow:

1. Installs pnpm and Node.js 24.
2. Installs dependencies.
3. Runs photo standardization.
4. Builds the photo manifest.
5. Builds the web app with `SKIP_PRECHECK=1`.
6. Copies `sitemap.xml` to `googlesitemap.xml` when present.
7. Syncs `apps/web/dist/` into the root `web/` directory and commits generated changes.
8. Deploys `apps/web/dist/` to GitHub Pages.

The committed `web/` directory can also be used as a static output target for platforms such as Cloudflare Pages.
